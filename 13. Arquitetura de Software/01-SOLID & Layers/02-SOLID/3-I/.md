# Inteface Segregation Principle (ISP)
### Principio de Segregação de Interface

Definição clássica:

    Nenhum cliente deve ser forçado a depender de métodos que ele não utiliza.

Define que se uma interface pode ser dividida em partes, então ela deve ser dividida. Isso evita interfaces grandes e muito poluídas, e garante interfaces específicas para clientes específicos.
Isso é bom para podermos definir o que queremos entregar, mesmo com todas as responsabilidades divididas, nossa classe principal pode herdar de todas elas para ter todos os métodos desejados, mas quando precisarmos passar nossa classe principal para outro método, não precisamos passar todos os métodos dela, podemos passar o tipo da interface que contém as funcionalidades que queremos e então nao vamos ter todos os métodos, e sim só aqueles que queremos no momento.
interfaces devem ser coesas por grupo.
