# Open Closes Principle (OCP)
### Princípio do Aberto-Fechado

Definição clássica: 

    "Uma classe deve estar aberta para extensão e fechada para modificação"

Isso significa que o comportamento de uma classe não deve ser acoplado dentro dela para além da regra de negócio, ou seja, não deve-se ter nada fixo que caso precise ser alterado terá que acessar o código original para realizar a modificação, o que implica na possibilidade de quebrar outras funcionalidades que antes estavam funcionando corretamente. Portanto, deve estar aberto para extensão, mas fechado para modificação direta. Não significa nunca modificar código, e sim evitar modificar código estável.
