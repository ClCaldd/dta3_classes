# Camadas

Conforme um sistema cresce, a ausência de separação de responsabilidades tende 
a aumentar o acoplamento e dificultar a manutenção, se o sistema do banco de 
dados (MySQL) passou a apresentar falhas de segurança e precisamos alterar para 
SQLServer seria necessário fazer alterações por toda parte do projeto. Com isso 
surgiu a necessidade de organizar melhor os projetos e declarar responsabilidades 
para cada parte, para que caso eu queira alterar o banco de dados eu não precise 
me preocupar em acabar quebrando a regra de negócio da minha aplicação.

    Isso sendo aplicado demonstra quais principios de SOLID que vimos anteriormente?


Pensando dessa forma, de que maneira deveriamos dividir nossa aplicação?
